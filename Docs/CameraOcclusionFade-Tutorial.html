Camera Occlusion Fade

Guia passo a passo para fazer fade em objetos que bloqueiam a câmara — inclui explicação linha a linha do shader AF/OcclusionFade para quem nunca escreveu HLSL.

Goal

When a wall, tree, or prop sits between the camera and the player, instead of hiding it completely, we swap its materials to a semi-transparent version. When it stops blocking, we put the original materials back.

1. The big picture

Every frame, the script does three things:

Camera ──raycast──▶ Player │ └── hits occluders (walls, props on a specific layer) For each hit renderer → save original materials, apply fade materials For renderers no longer hit → restore original materials

This runs in LateUpdate() so it happens after the player and camera have moved for the frame — you always fade based on the latest positions.

2. Key Unity concepts you'll use

ConceptWhat it means here
RaycastAllShoots a ray from camera toward player and returns every collider along the way (not just the first).
LayerMaskLets you choose which layers count as occluders (e.g. "Environment" only, not the player).
RendererThe component that actually draws a mesh. You need this to change materials.
sharedMaterialsThe asset materials on a renderer. Reading these gives you the originals without creating copies.
materialsSetting this creates instances at runtime — perfect for temporary fade swaps.
DictionaryTracks which renderers are currently faded and stores their original + fade material arrays.
HashSetStores "which renderers are blocking this frame" for fast lookup.
Important

new Material(...) allocates memory. Always Destroy() fade materials when you restore a renderer, or you'll leak memory over time.

3. Build order — type it in this sequence

  1. Cria o shader de fade (AF/OcclusionFade) — secção 4 + walkthrough 4b.
  2. Create the C# script skeleton with public fields and the inner OccludedRenderer class.
  3. Add Awake() to auto-create a fade material if none is assigned in the Inspector.
  4. Write LateUpdate() — raycast, collect hits, fade new ones, restore old ones.
  5. Write ApplyFade, RestoreRenderer, and RestoreAll.
  6. Write CreateFadeMaterial and the helpers that copy textures/colors from the original.
  7. Attach the script to your camera, assign Player + Occluder Layer, test in Play mode.

4. O shader de fade (URP)

Este shader desenha uma textura multiplicada por uma cor, com mistura alpha padrão. O valor _BaseColor.a é o que o torna semi-transparente — perfeito para paredes que bloqueiam a câmara.

Tags que valem a pena conhecer:

  • Queue = Transparent — desenha depois dos objetos opacos; obrigatório para ver através.
  • Blend SrcAlpha OneMinusSrcAlpha — mistura alpha clássica (como Photoshop em modo Normal).
  • ZWrite Off — objetos transparentes normalmente não escrevem no depth buffer (evita bugs de ordenação).

Guarda em Assets/Shaders/AF_OcclusionFade.shader:

Shader "AF/OcclusionFade"
{
    Properties
    {
        [MainTexture] _BaseMap("Albedo", 2D) = "white" {}
        [MainColor] _BaseColor("Color", Color) = (1, 1, 1, 0.25)
    }

    SubShader
    {
        Tags
        {
            "RenderType" = "Transparent"
            "Queue" = "Transparent"
            "RenderPipeline" = "UniversalPipeline"
            "IgnoreProjector" = "True"
        }

        Pass
        {
            Name "ForwardLit"
            Tags { "LightMode" = "UniversalForward" }

            Blend SrcAlpha OneMinusSrcAlpha
            ZWrite Off
            Cull Back

            HLSLPROGRAM
            #pragma vertex vert
            #pragma fragment frag
            #pragma multi_compile_fog

            #include "Packages/com.unity.render-pipelines.universal/ShaderLibrary/Core.hlsl"

            struct Attributes
            {
                float4 positionOS : POSITION;
                float2 uv : TEXCOORD0;
            };

            struct Varyings
            {
                float4 positionCS : SV_POSITION;
                float2 uv : TEXCOORD0;
                float fogFactor : TEXCOORD1;
            };

            TEXTURE2D(_BaseMap);
            SAMPLER(sampler_BaseMap);

            CBUFFER_START(UnityPerMaterial)
                float4 _BaseMap_ST;
                half4 _BaseColor;
            CBUFFER_END

            Varyings vert(Attributes input)
            {
                Varyings output;
                output.positionCS = TransformObjectToHClip(input.positionOS.xyz);
                output.uv = TRANSFORM_TEX(input.uv, _BaseMap);
                output.fogFactor = ComputeFogFactor(output.positionCS.z);
                return output;
            }

            half4 frag(Varyings input) : SV_Target
            {
                half4 col = SAMPLE_TEXTURE2D(_BaseMap, sampler_BaseMap, input.uv) * _BaseColor;
                col.rgb = MixFog(col.rgb, input.fogFactor);
                return col;
            }
            ENDHLSL
        }
    }
}

4b. Nunca escreveste um shader? Vamos linha a linha 🎭

Metáfora do teatro

Imagina que cada pixel do ecrã é um lugar na plateia. O vertex shader (vert) é o stage manager: coloca cada vértice do modelo 3D no sítio certo do palco (ecrã). O fragment shader (frag) é o pintor: decide a cor exacta de cada lugar. O shader inteiro é o guião desta peça de 73 linhas.

Mesh 3D → vert (posição + UV) → rasterização (GPU) → frag (cor + alpha) → ecrã

Acto I — A ficha de identidade

  1. 1
    Shader "AF/OcclusionFade"
    O nome na porta. É assim que o Unity encontra o shader: Shader.Find("AF/OcclusionFade") no teu script C#. AF é o teu "estúdio"; OcclusionFade é o papel.
  2. 2–7
    Properties { _BaseMap, _BaseColor }
    A lista de ingredientes que aparece no Inspector do Material. _BaseMap = imagem (albedo). _BaseColor = tinta por cima. O alpha 0.25 significa "75% transparente" — a parede fica fantasma mas ainda visível. Os atributos [MainTexture] e [MainColor] dizem ao URP "isto é a textura/cor principal".

Acto II — Regras de cenário

  1. 9
    SubShader
    Um bloco de instruções. Podes ter vários SubShaders para GPUs diferentes; aqui só precisas de um.
  2. 11–17
    Tags { RenderType, Queue, RenderPipeline… }
    Etiquetas para o motor de render:
    • Transparent = "não sou parede sólida, desenha-me por cima do chão mas mistura com o que está atrás"
    • UniversalPipeline = "só funciono no URP" (não no Built-in antigo)
    • IgnoreProjector = projectores de luz não me rebentam a transparência
  3. 19
    Pass
    Uma ida ao palco. Shaders complexos têm vários passes (sombra, profundidade, cor). O nosso tem um só: "desenha a cor".
  4. 21–22
    Name "ForwardLit" / LightMode = UniversalForward
    Diz ao URP: "sou um pass de iluminação forward normal" — o caminho standard para objetos visíveis à luz.
  5. 24–26
    Blend … / ZWrite Off / Cull Back
    As regras de mistura:
    • Blend SrcAlpha OneMinusSrcAlpha — fórmula mágica: cor final = cor do objeto × alpha + cor de trás × (1−alpha). É o "vidro fosco".
    • ZWrite Off — não gravas profundidade; se gravasses, objetos atrás de ti desapareciam de forma estranha.
    • Cull Back — não desenhas a cara de dentro do modelo (performance + evita artefactos).

Acto III — O guião HLSL

  1. 28
    HLSLPROGRAM
    Aqui começa código real (HLSL = linguagem de shaders da Microsoft, tipo C mas para GPUs). Tudo até ENDHLSL corre na placa gráfica, milhares de vezes por frame.
  2. 29–31
    #pragma vertex vert / fragment frag / multi_compile_fog
    Contratos com a GPU:
    • vert = função para cada vértice (cantos do mesh)
    • frag = função para cada pixel
    • multi_compile_fog = suporte opcional à névoa do Unity (liga/desliga variantes automaticamente)
  3. 33
    #include Core.hlsl
    Importas a caixa de ferramentas do URP: funções como TransformObjectToHClip, TRANSFORM_TEX, MixFog. Não reinventas a roda.
  4. 35–38
    struct Attributes { positionOS, uv }
    O pacote de entrada do mesh. Cada vértice traz:
    • positionOS — posição no espaço do Objecto (Space)
    • uv — coordenadas na textura (como GPS na imagem: 0,0 = canto, 1,1 = canto oposto)
  5. 40–44
    struct Varyings { positionCS, uv, fogFactor }
    O pacote de saída do vertex shader, que o fragment shader vai receber. positionCS = posição no espaço do Clip (ecrã). Os números em TEXCOORD0/1 são "correios internos" da GPU para passar dados entre estágios.
  6. 46–47
    TEXTURE2D(_BaseMap) / SAMPLER(sampler_BaseMap)
    Declaras a imagem e o modo de leitura (filtro, wrap). No URP moderno separas textura e sampler — é o padrão DX12/Vulkan.
  7. 49–52
    CBUFFER_START(UnityPerMaterial) … CBUFFER_END
    Variáveis que o Material controla e que a GPU consegue ler de forma eficiente:
    • _BaseMap_ST — tiling (repetir) e offset da textura (Unity cria isto automaticamente)
    • _BaseColor — a tinta + alpha que o teu script C# altera com SetColor("_BaseColor", …)

Acto IV — O stage manager: vert()

  1. 54–60
    Varyings vert(Attributes input) { … }
    Corre uma vez por vértice do modelo. Uma parede com 8 cantos? 8 chamadas. Um orc com 5000 vértices? 5000 chamadas — mas a GPU faz tudo em paralelo, ridiculamente rápido.

    Linha 56: TransformObjectToHClip — a viagem épica: espaço do objeto → mundo → câmara → ecrã normalizado. É o "onde fica este canto no ecrã?"
    Linha 57: TRANSFORM_TEX — aplica scale/offset do Inspector às UVs.
    Linha 58: ComputeFogFactor — guarda quanto névoa aplicar mais tarde, baseado na profundidade Z.

Acto V — O pintor: frag()

  1. 62–67
    half4 frag(Varyings input) : SV_Target
    Corre uma vez por pixel visível do objeto. Uma parede a ocupar metade do ecrã em 1080p? Centenas de milhares de chamadas — outra vez, a GPU adora isto.

    Linha 64: SAMPLE_TEXTURE2D(…, input.uv) — lê a cor da textura naquele ponto UV. Multiplica por _BaseColor — se a textura for tijolo vermelho e _BaseColor for branco semi-transparente, ficas com tijolo fantasma.
    Linha 65: MixFog — mistura névoa de cena se tiveres Fog activo.
    Linha 66: return col — entrega o pixel. O : SV_Target significa "isto vai para o framebuffer" (o ecrã).
  2. 69–72
    ENDHLSL / } / }
    Fim do código GPU, fim do Pass, fim do SubShader, fim do Shader. Cortina. 🎬
Como isto liga ao teu jogo

O script C# não mexe no shader directamente. Cria Material com este shader, copia a textura da parede original para _BaseMap, e põe _BaseColor.a = fadeAlpha (ex: 0.25). O shader faz o resto — cada frame, cada pixel, a GPU executa vert e frag e a parede fica translúcida entre a câmara e o jogador.

Se mudares isto no Material…O que vês
_BaseColor.a → 0Objecto invisível (só o collider continua a bloquear o raycast!)
_BaseColor.a → 1Objecto totalmente opaco (deixa de ser fade)
_BaseMap sem texturaQuadrado branco semi-transparente (default "white")
Apagas ZWrite OffProvavelmente bugs visuais com outros transparentes — não recomendado
Primeiro shader? Parabéns.

73 linhas e já tens transparência, texturas, fog e compatibilidade URP. A maioria dos shaders de jogos são variações deste esqueleto: Properties → Tags → Pass → vert → frag. Quando quiseres evoluir, experimenta mudar só a linha 64 — por exemplo multiplicar por um verde para debug: col.rgb *= half3(0,1,0);

5. O script C# — secção a secção

5a. Fields and tracking data

Public fields show up in the Inspector. The dictionary remembers every renderer you've faded this session.

namespace AF
{
    using UnityEngine;
    using System.Collections.Generic;

    public class CameraOcclusionFade : MonoBehaviour
    {
        public Transform player;
        public LayerMask occluderLayer;

        public Material fadeMaterial;   // template — shader = AF/OcclusionFade
        [Range(0f, 1f)]
        public float fadeAlpha = 0.25f; // 0 = invisible, 1 = solid

        // Key = renderer, Value = saved originals + runtime fade copies
        private readonly Dictionary<Renderer, OccludedRenderer> fadedRenderers = new();

        private class OccludedRenderer
        {
            public Material[] originals;
            public Material[] fades;
        }

5b. Awake — fallback material

If you forget to drag a material into the Inspector, the script finds the shader by name and creates one. Handy during prototyping.

        void Awake()
        {
            if (fadeMaterial == null)
            {
                Shader shader = Shader.Find("AF/OcclusionFade");
                if (shader != null)
                    fadeMaterial = new Material(shader);
            }
        }

5c. LateUpdate — the main loop

This is the heart of the system. Read it in four chunks:

Chunk 1 — build the ray

        void LateUpdate()
        {
            if (player == null || fadeMaterial == null) return;

            Vector3 dir = player.position - transform.position;
            float dist = dir.magnitude;

            Ray ray = new Ray(transform.position, dir);
            RaycastHit[] hits = Physics.RaycastAll(ray, dist, occluderLayer);

transform.position is the camera (this script lives on the camera). The ray only travels as far as the player — no need to hit things behind them.

Chunk 2 — fade everything currently blocking

            var currentHits = new HashSet<Renderer>();

            foreach (RaycastHit hit in hits)
            {
                Renderer rend = hit.collider.GetComponent<Renderer>();
                if (rend == null) continue;

                currentHits.Add(rend);
                ApplyFade(rend);
            }

Not every collider has a renderer (invisible trigger volumes, for example). Skip those.

Chunk 3 — find renderers that stopped blocking

            var toRestore = new List<Renderer>();

            foreach (var kvp in fadedRenderers)
            {
                if (kvp.Key == null)          // object was destroyed
                {
                    DestroyFadeMaterials(kvp.Value);
                    continue;
                }

                if (!currentHits.Contains(kvp.Key))
                    toRestore.Add(kvp.Key);
            }

            foreach (Renderer rend in toRestore)
                RestoreRenderer(rend);
        }

Compare "who we faded before" against "who is blocking now." Anyone in the old set but not the new set gets restored.

5d. Cleanup on disable/destroy

If you leave Play mode or disable the camera while objects are faded, you must put materials back and destroy runtime copies.

        void OnDisable() => RestoreAll();
        void OnDestroy() => RestoreAll();

5e. ApplyFade — swap to fade materials

        void ApplyFade(Renderer rend)
        {
            if (fadedRenderers.ContainsKey(rend)) return; // already faded

            Material[] originals = rend.sharedMaterials;
            var fades = new Material[originals.Length];

            for (int i = 0; i < originals.Length; i++)
                fades[i] = CreateFadeMaterial(originals[i]);

            fadedRenderers[rend] = new OccludedRenderer
            {
                originals = originals,
                fades = fades
            };

            rend.materials = fades;  // assign runtime instances
        }

Multi-material meshes (e.g. separate slots for body and trim) get one fade material per slot.

5f. RestoreRenderer — put originals back

        void RestoreRenderer(Renderer rend)
        {
            if (!fadedRenderers.TryGetValue(rend, out OccludedRenderer state)) return;

            rend.sharedMaterials = state.originals;
            DestroyFadeMaterials(state);
            fadedRenderers.Remove(rend);
        }

        void RestoreAll()
        {
            foreach (var kvp in fadedRenderers)
            {
                if (kvp.Key != null)
                    kvp.Key.sharedMaterials = kvp.Value.originals;

                DestroyFadeMaterials(kvp.Value);
            }

            fadedRenderers.Clear();
        }

5g. CreateFadeMaterial — copy look, lower alpha

Clone the template fade material, copy the original's texture and color, then override alpha.

        Material CreateFadeMaterial(Material original)
        {
            Material fade = new Material(fadeMaterial);
            CopyVisuals(original, fade);

            Color color = GetSourceColor(original);
            color.a = fadeAlpha;
            fade.SetColor("_BaseColor", color);

            return fade;
        }

5h. Helpers — support different shader property names

URP uses _BaseMap / _BaseColor. Older shaders use _MainTex / _Color. Your Synty/Lowpoly assets use _Lowpoly_DiffuseMap. Check which properties exist before reading them.

        static void CopyVisuals(Material source, Material target)
        {
            if (source == null) return;

            if (source.HasProperty("_BaseMap"))
            {
                target.SetTexture("_BaseMap", source.GetTexture("_BaseMap"));
                target.SetTextureScale("_BaseMap", source.GetTextureScale("_BaseMap"));
                target.SetTextureOffset("_BaseMap", source.GetTextureOffset("_BaseMap"));
            }
            else if (source.HasProperty("_MainTex"))
            {
                target.SetTexture("_BaseMap", source.GetTexture("_MainTex"));
                target.SetTextureScale("_BaseMap", source.GetTextureScale("_MainTex"));
                target.SetTextureOffset("_BaseMap", source.GetTextureOffset("_MainTex"));
            }
            else if (source.HasProperty("_Lowpoly_DiffuseMap"))
            {
                target.SetTexture("_BaseMap", source.GetTexture("_Lowpoly_DiffuseMap"));
            }
        }

        static Color GetSourceColor(Material source)
        {
            if (source.HasProperty("_BaseColor"))
                return source.GetColor("_BaseColor");
            if (source.HasProperty("_Color"))
                return source.GetColor("_Color");
            if (source.HasProperty("_Lowpoly_DiffuseColor"))
                return source.GetColor("_Lowpoly_DiffuseColor");
            return Color.white;
        }

        static void DestroyFadeMaterials(OccludedRenderer state)
        {
            if (state.fades == null) return;

            foreach (Material mat in state.fades)
            {
                if (mat != null)
                    Destroy(mat);
            }
        }
    }
}

6. Full script (copy reference)

Once you've typed each section, compare against the complete file:

namespace AF
{
    using UnityEngine;
    using System.Collections.Generic;

    public class CameraOcclusionFade : MonoBehaviour
    {
        public Transform player;
        public LayerMask occluderLayer;

        [Tooltip("Template material using AF/OcclusionFade. Textures are copied from each occluder at runtime.")]
        public Material fadeMaterial;

        [Range(0f, 1f)]
        public float fadeAlpha = 0.25f;

        private readonly Dictionary<Renderer, OccludedRenderer> fadedRenderers = new();

        private class OccludedRenderer
        {
            public Material[] originals;
            public Material[] fades;
        }

        void Awake()
        {
            if (fadeMaterial == null)
            {
                Shader shader = Shader.Find("AF/OcclusionFade");
                if (shader != null)
                    fadeMaterial = new Material(shader);
            }
        }

        void LateUpdate()
        {
            if (player == null || fadeMaterial == null) return;

            Vector3 dir = player.position - transform.position;
            float dist = dir.magnitude;

            Ray ray = new Ray(transform.position, dir);
            RaycastHit[] hits = Physics.RaycastAll(ray, dist, occluderLayer);

            var currentHits = new HashSet<Renderer>();

            foreach (RaycastHit hit in hits)
            {
                Renderer rend = hit.collider.GetComponent<Renderer>();
                if (rend == null) continue;

                currentHits.Add(rend);
                ApplyFade(rend);
            }

            var toRestore = new List<Renderer>();

            foreach (var kvp in fadedRenderers)
            {
                if (kvp.Key == null)
                {
                    DestroyFadeMaterials(kvp.Value);
                    continue;
                }

                if (!currentHits.Contains(kvp.Key))
                    toRestore.Add(kvp.Key);
            }

            foreach (Renderer rend in toRestore)
                RestoreRenderer(rend);
        }

        void OnDisable() => RestoreAll();
        void OnDestroy() => RestoreAll();

        void ApplyFade(Renderer rend)
        {
            if (fadedRenderers.ContainsKey(rend)) return;

            Material[] originals = rend.sharedMaterials;
            var fades = new Material[originals.Length];

            for (int i = 0; i < originals.Length; i++)
                fades[i] = CreateFadeMaterial(originals[i]);

            fadedRenderers[rend] = new OccludedRenderer
            {
                originals = originals,
                fades = fades
            };

            rend.materials = fades;
        }

        void RestoreRenderer(Renderer rend)
        {
            if (!fadedRenderers.TryGetValue(rend, out OccludedRenderer state)) return;

            rend.sharedMaterials = state.originals;
            DestroyFadeMaterials(state);
            fadedRenderers.Remove(rend);
        }

        void RestoreAll()
        {
            foreach (var kvp in fadedRenderers)
            {
                if (kvp.Key != null)
                    kvp.Key.sharedMaterials = kvp.Value.originals;

                DestroyFadeMaterials(kvp.Value);
            }

            fadedRenderers.Clear();
        }

        Material CreateFadeMaterial(Material original)
        {
            Material fade = new Material(fadeMaterial);
            CopyVisuals(original, fade);

            Color color = GetSourceColor(original);
            color.a = fadeAlpha;
            fade.SetColor("_BaseColor", color);

            return fade;
        }

        static void CopyVisuals(Material source, Material target)
        {
            if (source == null) return;

            if (source.HasProperty("_BaseMap"))
            {
                target.SetTexture("_BaseMap", source.GetTexture("_BaseMap"));
                target.SetTextureScale("_BaseMap", source.GetTextureScale("_BaseMap"));
                target.SetTextureOffset("_BaseMap", source.GetTextureOffset("_BaseMap"));
            }
            else if (source.HasProperty("_MainTex"))
            {
                target.SetTexture("_BaseMap", source.GetTexture("_MainTex"));
                target.SetTextureScale("_BaseMap", source.GetTextureScale("_MainTex"));
                target.SetTextureOffset("_BaseMap", source.GetTextureOffset("_MainTex"));
            }
            else if (source.HasProperty("_Lowpoly_DiffuseMap"))
            {
                target.SetTexture("_BaseMap", source.GetTexture("_Lowpoly_DiffuseMap"));
            }
        }

        static Color GetSourceColor(Material source)
        {
            if (source.HasProperty("_BaseColor"))
                return source.GetColor("_BaseColor");

            if (source.HasProperty("_Color"))
                return source.GetColor("_Color");

            if (source.HasProperty("_Lowpoly_DiffuseColor"))
                return source.GetColor("_Lowpoly_DiffuseColor");

            return Color.white;
        }

        static void DestroyFadeMaterials(OccludedRenderer state)
        {
            if (state.fades == null) return;

            foreach (Material mat in state.fades)
            {
                if (mat != null)
                    Destroy(mat);
            }
        }
    }
}

7. Unity setup checklist

  • Create a layer (e.g. Occluder) and assign it to walls/props that should fade.
  • Those objects need a Collider (raycasts hit colliders, not meshes directly).
  • Attach CameraOcclusionFade to your camera object.
  • Drag the Player transform into the script's Player field.
  • Set Occluder Layer to your occluder layer mask.
  • Optionally create a Material using shader AF/OcclusionFade and assign it as Fade Material.
  • Tweak Fade Alpha in Play mode until it feels right (~0.2–0.35).

8. Things to experiment with (good learning exercises)

  • Smooth fade — lerp fadeAlpha over time instead of snapping.
  • Spherecast — use Physics.SphereCastAll with a small radius so thin poles don't get missed.
  • MaterialPropertyBlock — avoid creating new Material instances; set alpha on a shared fade material per renderer (more advanced, better performance).
  • Dither fade — looks opaque-ish but clips pixels; avoids transparent sorting issues.
  • Debug draw — Debug.DrawRay(transform.position, dir, Color.red) to visualize the ray in the Scene view.

9. Common gotchas

ProblemLikely cause
Nothing fadesWrong layer mask, missing collider, or no Renderer on the object.
Faded object stays fadedRestore logic not running — check dictionary cleanup and OnDisable.
Pink materialsShader not found — make sure AF_OcclusionFade.shader compiled without errors.
Wrong texture on fadeOriginal shader uses a property name you didn't handle — inspect the material in Unity and add it to CopyVisuals.
Memory climbingFade materials not destroyed — verify DestroyFadeMaterials runs on restore.
