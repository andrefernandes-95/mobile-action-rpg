Menu UI · Arquitetura escalável

Redesenho da camada de menus do projeto AF (UI Toolkit). Substitui o GameMenuController monolítico por uma arquitetura desacoplada e reutilizável em Equipment, Shop e Settings. Continua o doc Soulslike · Economy · Menu · Dialogue — assume que já tens tudo antes do GameMenuController (economia, progressão, durabilidade, loja/ferreiro, consumíveis).

0. Porque mudar — o problema do god class

Sintomas do GameMenuController atual
  • Uma classe faz tudo — navegação, dados de inventário, loja, ferreiro, opções, construção de linhas, ações. Cresce sem limite.
  • switch por tab — cada nova secção é mais um case no mesmo método. Frágil e difícil de testar.
  • Acoplamento — o menu conhece InventoryManager, ShopService, BlacksmithService, SceneManager… tudo ao mesmo tempo.
  • Nada reutilizável — a lista de itens do Equipment não serve para a Shop sem copiar/colar.
Objetivos do redesenho
  • Cada ecrã (Equipment, Shop, Settings) é uma classe pequena e independente.
  • Abrir/fechar/pausar centralizado num sítio só — os ecrãs não sabem uns dos outros.
  • Tabs adicionam-se sem tocar em código existente (Open/Closed Principle).
  • A linha de item é um componente reutilizável (Equipment reusa em Shop).
  • Dependências injetadas por [SerializeField] — testável e explícito.

MenuManager

Único ponto que sabe que ecrã está ativo. Os ecrãs registam-se sozinhos no OnEnable. O HUD só pede Open(MenuId) — não conhece as classes concretas. Respeita GameFlow.IsInCutscene (do doc anterior).

Assets/Scripts/UI/Menu/MenuManager.cs
using System.Collections.Generic;
using UnityEngine;

namespace AF
{
    public class MenuManager : MonoBehaviour
    {
        public static MenuManager Instance { get; private set; }

        [SerializeField] bool pauseWhileOpen = true;

        readonly Dictionary<MenuId, MenuScreen> screens = new();
        MenuScreen active;

        public bool AnyOpen => active != null;
        public MenuId ActiveId => active != null ? active.Id : MenuId.None;

        void Awake()
        {
            if (Instance != null && Instance != this)
            {
                Destroy(gameObject);
                return;
            }
            Instance = this;
        }

        public void Register(MenuScreen screen) => screens[screen.Id] = screen;

        public void Unregister(MenuScreen screen)
        {
            if (screens.TryGetValue(screen.Id, out var s) && s == screen)
                screens.Remove(screen.Id);
        }

        public void Toggle(MenuId id)
        {
            if (active != null && active.Id == id) Close();
            else Open(id);
        }

        public void Open(MenuId id)
        {
            if (GameFlow.IsInCutscene) return;
            if (!screens.TryGetValue(id, out var next)) return;
            if (next == active) return;

            active?.Exit();
            active = next;
            active.Enter();

            if (pauseWhileOpen) Time.timeScale = 0f;
        }

        public void Close()
        {
            if (active == null) return;

            active.Exit();
            active = null;

            if (pauseWhileOpen) Time.timeScale = 1f;
        }
    }
}
Escalar para um stack

Se um dia quiseres sub-menus empilhados (ex.: confirmar compra por cima da loja), troca o campo active por um Stack<MenuScreen>: Open faz suspend do topo e push; Close faz pop e resume. O resto da API fica igual — os ecrãs não mudam.

3. Base — MenuScreen (Template Method)

Contrato comum

Todos os UXML de menu partilham a estrutura Container > CloseButton + Tab. A base trata do CloseButton, display e registo. As subclasses só implementam Bind() (querys uma vez) e opcionalmente OnOpen() / OnClose() (refresh de dados).

Assets/Scripts/UI/Menu/MenuScreen.cs
using UnityEngine;
using UnityEngine.UIElements;

namespace AF
{
    [RequireComponent(typeof(UIDocument))]
    public abstract class MenuScreen : MonoBehaviour
    {
        [SerializeField] protected UIDocument document;

        public abstract MenuId Id { get; }
        public bool IsOpen { get; private set; }

        protected VisualElement Root { get; private set; }

        protected virtual void Awake()
        {
            if (document == null) document = GetComponent<UIDocument>();
        }

        protected virtual void OnEnable()
        {
            Root = document.rootVisualElement;

            var closeButton = Root.Q<Button>("CloseButton");
            if (closeButton != null)
                closeButton.clicked += () => MenuManager.Instance?.Close();

            Bind();
            Root.style.display = DisplayStyle.None;
            MenuManager.Instance?.Register(this);
        }

        protected virtual void OnDisable()
        {
            MenuManager.Instance?.Unregister(this);
        }

        // Chamado só pelo MenuManager — direção única.
        internal void Enter()
        {
            Root.style.display = DisplayStyle.Flex;
            IsOpen = true;
            OnOpen();
        }

        internal void Exit()
        {
            OnClose();
            Root.style.display = DisplayStyle.None;
            IsOpen = false;
        }

        /// <summary>Query aos elementos, uma vez. Sem lógica de dados.</summary>
        protected abstract void Bind();

        /// <summary>Refresh sempre que abre.</summary>
        protected virtual void OnOpen() { }
        protected virtual void OnClose() { }
    }
}

4. HUD — ligar os botões do PlayerHUD

Desacoplado

O PlayerHUD.uxml tem Equipment, Shop, SettingsButton. Um componente único liga-os ao MenuManager por MenuId — não conhece nenhuma classe de ecrã.

Assets/Scripts/UI/Menu/HudMenuButtons.cs
using UnityEngine;
using UnityEngine.UIElements;

namespace AF
{
    public class HudMenuButtons : MonoBehaviour
    {
        [SerializeField] UIDocument hudDocument;

        void OnEnable()
        {
            var root = hudDocument.rootVisualElement;
            Bind(root, "Equipment", MenuId.Equipment);
            Bind(root, "Shop", MenuId.Shop);
            Bind(root, "SettingsButton", MenuId.Settings);
        }

        void Bind(VisualElement root, string buttonName, MenuId id)
        {
            var btn = root.Q<Button>(buttonName);
            if (btn != null)
                btn.clicked += () => MenuManager.Instance.Toggle(id);
        }
    }
}

5. Tabs reutilizáveis — ITabPage + TabView (Strategy)

Uma tab = uma estratégia

Cada categoria (Weapons, Shields…) implementa ITabPage: sabe a que botão responde e como construir o seu conteúdo num host. O TabView liga botões ↔ páginas e troca a ativa. Adicionar uma tab nova = criar uma classe e registá-la; zero switch.

Assets/Scripts/UI/Menu/ITabPage.cs
using UnityEngine.UIElements;

namespace AF
{
    public interface ITabPage
    {
        /// <summary>Nome do Button no UXML que ativa esta página.</summary>
        string TabButtonName { get; }

        /// <summary>Constrói / refresca o conteúdo dentro do host.</summary>
        void Enter(VisualElement host);

        /// <summary>Limpeza opcional ao sair da tab.</summary>
        void Leave();
    }
}
Assets/Scripts/UI/Menu/TabView.cs
using System.Collections.Generic;
using UnityEngine.UIElements;

namespace AF
{
    public class TabView
    {
        readonly VisualElement buttonsRoot;
        readonly VisualElement contentHost;
        readonly List<ITabPage> pages = new();

        ITabPage current;

        public TabView(VisualElement buttonsRoot, VisualElement contentHost)
        {
            this.buttonsRoot = buttonsRoot;
            this.contentHost = contentHost;
        }

        public void AddPage(ITabPage page)
        {
            pages.Add(page);

            var btn = buttonsRoot.Q<Button>(page.TabButtonName);
            if (btn != null)
                btn.clicked += () => Select(page);
        }

        public void Select(ITabPage page)
        {
            if (current == page)
            {
                page.Enter(contentHost); // refresh
                return;
            }

            current?.Leave();
            current = page;
            contentHost.Clear();
            current.Enter(contentHost);
        }

        public void SelectFirst()
        {
            if (pages.Count > 0) Select(pages[0]);
        }
    }
}

6. Linhas de lista reutilizáveis — ItemRowView (MVVM-lite)

Uma linha, muitos usos

A linha deixa de ter os botões fixos Equip/Repair/Sell no UXML. Passa a receber um modelo (nome, ícone, descrição, estado, indicador e uma lista de ações). Assim a mesma linha serve o Equipment (Equip/Repair/Sell) e a Shop (Buy) sem duplicação.

Assets/Scripts/UI/Menu/ItemRowModel.cs
using System;
using System.Collections.Generic;
using UnityEngine;
using UnityEngine.UIElements;

namespace AF
{
    public struct ItemAction
    {
        public string Label;
        public Action OnClick;
        public bool Enabled;

        public ItemAction(string label, Action onClick, bool enabled = true)
        {
            Label = label;
            OnClick = onClick;
            Enabled = enabled;
        }
    }

    public class ItemRowModel
    {
        public string Name;
        public Sprite Icon;
        public string Description;          // ex.: "Attack Damage: 110"
        public string Status;               // ex.: "Current Durability: 100/200"
        public bool ShowIndicator;
        public Color IndicatorColor = Color.white;
        public readonly List<ItemAction> Actions = new();
    }
}
Assets/Scripts/UI/Menu/ItemRowView.cs
using UnityEngine;
using UnityEngine.UIElements;

namespace AF
{
    /// <summary>
    /// Constrói uma linha a partir do template WeaponListItem.uxml + um modelo.
    /// Os nomes batem com o teu UXML: ItemName, ItemIcon, Description, Status,
    /// Indicator, Actions.
    /// </summary>
    public static class ItemRowView
    {
        public static VisualElement Build(VisualTreeAsset template, ItemRowModel model)
        {
            VisualElement row = template.Instantiate();

            var name = row.Q<Label>("ItemName");
            if (name != null) name.text = model.Name;

            var icon = row.Q("ItemIcon");
            if (icon != null && model.Icon != null)
                icon.style.backgroundImage = new StyleBackground(model.Icon);

            var description = row.Q<Label>("Description");
            if (description != null) description.text = model.Description;

            var status = row.Q<Label>("Status");
            if (status != null) status.text = model.Status;

            var indicator = row.Q("Indicator");
            if (indicator != null)
            {
                indicator.style.display = model.ShowIndicator
                    ? DisplayStyle.Flex : DisplayStyle.None;
                indicator.style.backgroundColor = model.IndicatorColor;
            }

            var actions = row.Q("Actions");
            if (actions != null)
            {
                actions.Clear();
                foreach (var action in model.Actions)
                {
                    var captured = action;
                    var btn = new Button(() => captured.OnClick?.Invoke())
                    {
                        text = captured.Label
                    };
                    btn.AddToClassList("button");
                    btn.SetEnabled(captured.Enabled);
                    actions.Add(btn);
                }
            }

            return row;
        }
    }
}

7. EquipmentScreen + tabs

Ecrã fino

O EquipmentScreen só monta o TabView e regista as páginas. A lógica de armas vive em WeaponsTab. Categorias sem dados ainda usam ComingSoonTab.

Assets/Scripts/UI/Menu/Equipment/EquipmentScreen.cs
using UnityEngine;
using UnityEngine.UIElements;

namespace AF
{
    public class EquipmentScreen : MenuScreen
    {
        public override MenuId Id => MenuId.Equipment;

        [Header("Template")]
        [SerializeField] VisualTreeAsset weaponRowTemplate; // WeaponListItem.uxml

        [Header("Systems")]
        [SerializeField] InventoryManager inventory;
        [SerializeField] EquipmentManager equipment;
        [SerializeField] BlacksmithService blacksmith;
        [SerializeField] ShopService shop;

        TabView tabs;

        protected override void Bind()
        {
            var categories = Root.Q("EquipmentCategories");
            var host = Root.Q("TabContentHost");

            tabs = new TabView(categories, host);
            tabs.AddPage(new WeaponsTab(weaponRowTemplate, inventory, equipment, blacksmith, shop));
            tabs.AddPage(new ComingSoonTab("ShieldsTabButton", "Shields — brevemente"));
            tabs.AddPage(new ComingSoonTab("ArmorsTabButton", "Armors — brevemente"));
            tabs.AddPage(new ComingSoonTab("Accessory1TabButton", "Accessory 1 — brevemente"));
            tabs.AddPage(new ComingSoonTab("Accessory2TabButton", "Accessory 2 — brevemente"));
        }

        protected override void OnOpen() => tabs.SelectFirst();
    }
}
Assets/Scripts/UI/Menu/Equipment/WeaponsTab.cs
using UnityEngine;
using UnityEngine.UIElements;

namespace AF
{
    public class WeaponsTab : ITabPage
    {
        public string TabButtonName => "WeaponsTabButton";

        readonly VisualTreeAsset rowTemplate;
        readonly InventoryManager inventory;
        readonly EquipmentManager equipment;
        readonly BlacksmithService blacksmith;
        readonly ShopService shop;

        VisualElement host;

        public WeaponsTab(
            VisualTreeAsset rowTemplate,
            InventoryManager inventory,
            EquipmentManager equipment,
            BlacksmithService blacksmith,
            ShopService shop)
        {
            this.rowTemplate = rowTemplate;
            this.inventory = inventory;
            this.equipment = equipment;
            this.blacksmith = blacksmith;
            this.shop = shop;
        }

        public void Enter(VisualElement contentHost)
        {
            host = contentHost;
            host.Clear();

            int equippedDamage = equipment != null && equipment.weaponInstance != null
                ? equipment.weaponInstance.weaponData.amount
                : 0;

            var header = new Label($"Current Attack Damage: {equippedDamage}");
            header.AddToClassList("label");
            host.Add(header);

            var scroll = new ScrollView { style = { marginTop = 16 } };
            host.Add(scroll);

            if (inventory == null) return;

            foreach (var wi in inventory.ownedWeapons)
                scroll.Add(ItemRowView.Build(rowTemplate, BuildModel(wi, equippedDamage)));
        }

        public void Leave() { }

        ItemRowModel BuildModel(WeaponInstance wi, int equippedDamage)
        {
            Weapon data = wi.weaponData;
            bool equipped = equipment != null && equipment.weaponInstance == wi;

            var model = new ItemRowModel
            {
                Name = data.displayName,
                Icon = data.icon,
                Description = $"Attack Damage: {data.amount}",
                Status = $"Current Durability: {wi.durability}/{wi.maxDurability}",
                ShowIndicator = true,
                IndicatorColor = data.amount >= equippedDamage
                    ? new Color(0.23f, 1f, 0.05f)   // verde: melhor/igual
                    : new Color(0.90f, 0.40f, 0.40f) // vermelho: pior
            };

            if (equipped)
            {
                model.Actions.Add(new ItemAction("Equipped", null, enabled: false));
            }
            else
            {
                model.Actions.Add(new ItemAction("Equip", () =>
                {
                    equipment.EquipWeapon(data);
                    Enter(host); // refresh
                }));
            }

            int repairCost = wi.RepairCost;
            model.Actions.Add(new ItemAction(
                repairCost > 0 ? $"Repair ({repairCost} Gold)" : "Repair",
                () => { if (blacksmith.Repair(wi)) Enter(host); },
                enabled: repairCost > 0));

            int sellValue = shop != null ? shop.SellValue(data) : 0;
            model.Actions.Add(new ItemAction($"Sell ({sellValue} Gold)",
                () => { if (shop.SellWeapon(wi)) Enter(host); }));

            return model;
        }
    }
}

8. Reutilizar em Shop e Settings

O padrão paga-se aqui

A Shop reusa MenuScreen + TabView + ItemRowView. A Settings é trivial — mostra que a base serve até menus simples.

using UnityEngine;

namespace AF
{
    public class ShopScreen : MenuScreen
    {
        public override MenuId Id => MenuId.Shop;

        [SerializeField] VisualTreeAsset weaponRowTemplate;
        [SerializeField] ShopService shop;

        TabView tabs;

        protected override void Bind()
        {
            var categories = Root.Q("EquipmentCategories"); // ou "ShopCategories"
            var host = Root.Q("TabContentHost");

            tabs.AddPage(new BuyWeaponsTab(weaponRowTemplate, shop));

    }

Assets/Scripts/UI/Menu/Shop/BuyWeaponsTab.cs
using UnityEngine;
using UnityEngine.UIElements;

namespace AF
{
    public class BuyWeaponsTab : ITabPage
    {
        public string TabButtonName => "WeaponsTabButton";

        readonly VisualTreeAsset rowTemplate;
        readonly ShopService shop;
        VisualElement host;

        public BuyWeaponsTab(VisualTreeAsset rowTemplate, ShopService shop)
        {
            this.rowTemplate = rowTemplate;
            this.shop = shop;
        }

        public void Enter(VisualElement contentHost)
        {
            host = contentHost;
            host.Clear();

            var scroll = new ScrollView();
            host.Add(scroll);

            if (shop == null || shop.Catalog == null) return;

            foreach (var weapon in shop.Catalog.weapons)
            {
                if (weapon == null) continue;

                var model = new ItemRowModel
                {
                    Name = weapon.displayName,
                    Icon = weapon.icon,
                    Description = $"Attack Damage: {weapon.amount}",
                    Status = $"Durability: {weapon.maxDurability}",
                    ShowIndicator = false
                };

                // mesma linha, ação diferente — sem duplicar UI
                model.Actions.Add(new ItemAction($"Buy ({weapon.buyPrice} Gold)",
                    () => { if (shop.BuyWeapon(weapon)) Enter(host); }));

                scroll.Add(ItemRowView.Build(rowTemplate, model));
            }
        }

        public void Leave() { }
    }
}
Assets/Scripts/UI/Menu/Settings/SettingsScreen.cs
using UnityEngine;
using UnityEngine.SceneManagement;
using UnityEngine.UIElements;

namespace AF
{
    public class SettingsScreen : MenuScreen
    {
        public override MenuId Id => MenuId.Settings;

        protected override void Bind()
        {
            var exit = Root.Q<Button>("ExitGameButton");
            if (exit != null) exit.clicked += QuitGame;

            // exemplo: botão de reiniciar, se o adicionares ao UXML
            var restart = Root.Q<Button>("RestartButton");
            if (restart != null) restart.clicked += Restart;
        }

        void Restart()
        {
            Time.timeScale = 1f;
            SceneManager.LoadScene(SceneManager.GetActiveScene().name);
        }

        void QuitGame()
        {
            Application.Quit();
#if UNITY_EDITOR
            UnityEditor.EditorApplication.isPlaying = false;
#endif
        }
    }
}
