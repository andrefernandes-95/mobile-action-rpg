Movimento analógico — refactor

Joystick pressure → velocidade do player + blend de Speed + velocidade de yaw da câmara. Código concreto para o teu PlayerInput, CharacterManager e ThirdPersonCamera.

0. O problema

O joystick já é analógico — o teu código é que não

O Joystick Pack devolve Horizontal/Vertical em [-1..1]. Quanto mais empurras, maior a magnitude. Mas no PlayerInput fazes isto:

Vector3 moveDir = camForward * v + camRight * h;

if (moveDir.magnitude > 0.1f)
    characterManager.Move(moveDir.normalized); // ← mata a pressão
else
    characterManager.Stop();
  • .normalized força magnitude = 1 → sempre a mesma velocidade.
  • Move só faz SetDestination → o NavMeshAgent corre sempre a agent.speed (full).
  • animator.SetFloat("Speed", agent.desiredVelocity.magnitude) → ou ~0 ou ~speed. Sem meio-termo no blend tree.
  • A câmara usa yawFollowSpeed constante → vira sempre ao mesmo ritmo.

Sim, dá para ter meio-termo. A pressão do stick (0..1) escala: velocidade do agent, parâmetro Speed da animação, e yaw follow da câmara.

1. Modelo alvo

Joystick.Direction (x,y) │ magnitude = clamp01(length) ← pressão 0..1 │ direction = normalized ▼ PlayerInput │ Move(dir, moveAmount) │ MoveAmount público (para a camara) ▼ CharacterManager │ agent.speed = walkSpeed + (runSpeed-walkSpeed)*amount │ SetDestination(...) │ animator Speed = amount (0 idle · 0.5 walk · 1 run) ▼ ThirdPersonCamera │ yawFollow = lerp(min, max, MoveAmount) │ posição segue como já faz

2. PlayerInput — preservar magnitude

Assets/Scripts/Core/PlayerInput.cs
namespace AF
{
    using UnityEngine;
    using UnityEngine.Events;

    public class PlayerInput : MonoBehaviour
    {
        Camera mainCam;
        public Joystick joystick;
        public CharacterManager startingCharacterManager;

        [Header("Current Character")]
        public CharacterManager characterManager;

        [Header("Analog")]
        [Tooltip("Abaixo disto = parado (deadzone).")]
        [SerializeField] float moveDeadzone = 0.12f;
        [Tooltip("Curva da pressão. 1 = linear. >1 = precisa empurrar mais para correr.")]
        [SerializeField] float inputExponent = 1.35f;

        [HideInInspector] public UnityEvent<CharacterManager> onSetCurrentCharacter;

        /// <summary>0..1 — pressão actual do stick. A câmara lê isto.</summary>
        public float MoveAmount { get; private set; }

        /// <summary>Direção horizontal normalizada (pode ser zero).</summary>
        public Vector3 MoveDirection { get; private set; }

        void OnEnable()
        {
            mainCam = Camera.main;
            SetCharacter(startingCharacterManager);
        }

        void SetCharacter(CharacterManager character)
        {
            if (characterManager != null)
                characterManager.GiveControlToAI();

            characterManager = character;
            characterManager.GiveControlToPlayer();
            onSetCurrentCharacter?.Invoke(characterManager);
        }

        void Update()
        {
            if (Input.GetKeyDown(KeyCode.Space))
            {
                OnSpecialAbility();
                return;
            }

            HandleMovement();
        }

        void HandleMovement()
        {
            MoveAmount = 0f;
            MoveDirection = Vector3.zero;

            if (mainCam == null || characterManager == null)
                return;

            float h = 0f;
            float v = 0f;

            if (joystick != null)
            {
                h += joystick.Horizontal;
                v += joystick.Vertical;
            }

            // Teclado no editor (também "analógico" via GetAxis, mas tipicamente -1/0/1)
            h += Input.GetAxis("Horizontal");
            v += Input.GetAxis("Vertical");

            Vector3 camForward = mainCam.transform.forward;
            Vector3 camRight = mainCam.transform.right;
            camForward.y = 0f;
            camRight.y = 0f;
            camForward.Normalize();
            camRight.Normalize();

            Vector3 raw = camForward * v + camRight * h;
            float magnitude = Mathf.Clamp01(raw.magnitude);

            if (magnitude < moveDeadzone)
            {
                characterManager.Stop();
                return;
            }

            // Remapeia deadzone→1 para 0→1 (evita salto ao sair da deadzone)
            float remapped = Mathf.InverseLerp(moveDeadzone, 1f, magnitude);
            float amount = Mathf.Pow(remapped, inputExponent);

            Vector3 dir = raw / magnitude; // normalized sem re-calcular

            MoveAmount = amount;
            MoveDirection = dir;

            characterManager.Move(dir, amount);
        }

        public void OnAttack()
        {
            characterManager?.combatManager.Attack();
        }

        public void OnSpecialAbility()
        {
            characterManager?.combatManager.OnSpecialAbility();
        }

        public Vector3 GetDodgeDirection(bool isLockedOn)
        {
            float h = (joystick != null ? joystick.Horizontal : 0f) + Input.GetAxis("Horizontal");
            float v = (joystick != null ? joystick.Vertical : 0f) + Input.GetAxis("Vertical");

            Vector3 camForward = mainCam.transform.forward * (isLockedOn ? -1f : 1f);
            Vector3 camRight = mainCam.transform.right;
            camForward.y = 0f;
            camRight.y = 0f;
            camForward.Normalize();
            camRight.Normalize();

            Vector3 dodgeDir = (camForward * v + camRight * h);
            if (dodgeDir.sqrMagnitude < 0.01f)
                return characterManager.transform.forward;

            return dodgeDir.normalized;
        }
    }
}
O que mudou
  • Já não se chama Move(dir.normalized) — passa-se amount.
  • MoveAmount fica público para a câmara.
  • inputExponent > 1 dá mais controlo na zona baixa (toques leves = walk).

3. CharacterManager — velocidade + Speed blend

Assets/Scripts/Core/CharacterManager.cs — secções relevantes
namespace AF
{
    using UnityEngine;
    using UnityEngine.AI;

    public class CharacterManager : MonoBehaviour
    {
        [Header("Name")]
        public new string name = "";

        [Header("Components")]
        public Animator animator;
        public NavMeshAgent agent;

        [Header("Character Components")]
        public Health health;
        public CharacterDamageReceiver characterDamageReceiver;
        public Dodge dodge;
        public Perception perception;
        public LockOn lockOn;
        public CombatManager combatManager;

        [Header("Locomotion")]
        public float rotationSpeed = 10f;
        [Tooltip("Velocidade com stick leve (walk).")]
        public float walkSpeed = 1.8f;
        [Tooltip("Velocidade com stick a fundo (run).")]
        public float runSpeed = 5.5f;
        [Tooltip("Damp do parâmetro Speed no Animator.")]
        public float animSpeedDamp = 0.12f;

        public bool isBusy = false;

        [Header("Patrol")]
        public Transform[] patrolPoints;

        float stoppingDistance;
        float baseAgentSpeed;

        /// <summary>0..1 — último input de movimento do player. Câmara / UI podem ler.</summary>
        public float MoveAmount { get; private set; }

        static readonly int SpeedHash = Animator.StringToHash("Speed");

        void Awake()
        {
            stoppingDistance = agent.stoppingDistance;
            baseAgentSpeed = agent.speed;
            // Se não configurares walk/run no inspector, deriva do agent.speed actual
            if (runSpeed <= 0f) runSpeed = baseAgentSpeed > 0f ? baseAgentSpeed : 5.5f;
            if (walkSpeed <= 0f) walkSpeed = runSpeed * 0.35f;
        }

        void Start()
        {
            agent.updateRotation = false;
        }

        public void OnIdle()
        {
            isBusy = false;
        }

        void Update()
        {
            UpdateRotation();
            UpdateAnimation();
        }

        /// <summary>
        /// direction = normalizado. moveAmount = 0..1 (pressão do stick).
        /// </summary>
        public void Move(Vector3 direction, float moveAmount = 1f)
        {
            if (isBusy) return;
            if (!agent.enabled || !agent.isOnNavMesh) return;

            MoveAmount = Mathf.Clamp01(moveAmount);

            agent.speed = Mathf.Lerp(walkSpeed, runSpeed, MoveAmount);

            // Destino curto à frente — o agent anda à velocidade que acabámos de setar
            float lookAhead = Mathf.Max(0.35f, agent.speed * 0.35f);
            Vector3 targetPos = transform.position + direction.normalized * lookAhead;
            agent.SetDestination(targetPos);
        }

        /// <summary>Compat: AI antiga que chama Move(dir) sem amount → full speed.</summary>
        public void Move(Vector3 direction)
        {
            Move(direction, 1f);
        }

        public void Stop()
        {
            MoveAmount = 0f;

            if (agent.enabled && agent.isOnNavMesh)
            {
                agent.ResetPath();
                agent.velocity = Vector3.zero;
            }
        }

        void UpdateRotation()
        {
            if (health.IsDead) return;

            if (lockOn != null && lockOn.isLockedOn && lockOn.lockOnTarget != null)
            {
                Vector3 dir = lockOn.lockOnTarget.position - agent.nextPosition;
                dir.y = 0f;
                if (dir.sqrMagnitude < 0.0001f) return;

                Quaternion targetRot = Quaternion.LookRotation(dir.normalized);
                float t = Mathf.Clamp01(lockOn.lockOnRotationSpeed * Time.deltaTime);
                transform.rotation = Quaternion.Slerp(transform.rotation, targetRot, t);
                return;
            }

            Vector3 moveDir = agent.desiredVelocity;
            if (moveDir.sqrMagnitude < 0.01f) return;

            Quaternion moveRot = Quaternion.LookRotation(moveDir.normalized);

            // Rodar mais depressa quando o stick está a fundo
            float rot = rotationSpeed * Mathf.Lerp(0.55f, 1.25f, MoveAmount);
            transform.rotation = Quaternion.Slerp(transform.rotation, moveRot, rot * Time.deltaTime);
        }

        void UpdateAnimation()
        {
            // Preferimos o input (estável) ao desiredVelocity (oscila com o NavMesh)
            float speedParam = MoveAmount;

            // Fallback para AI / quando MoveAmount não é setado por input
            if (!IsPlayer())
            {
                float max = Mathf.Max(0.01f, runSpeed);
                speedParam = Mathf.Clamp01(agent.desiredVelocity.magnitude / max);
            }

            animator.SetFloat(SpeedHash, speedParam, animSpeedDamp, Time.deltaTime);
        }

        public bool IsPlayer() => GameContext.Player == this;
        public CharacterManager GetPlayer() => GameContext.Player;

        public void GiveControlToAI()
        {
            agent.stoppingDistance = stoppingDistance;
            agent.speed = baseAgentSpeed > 0f ? baseAgentSpeed : runSpeed;
            MoveAmount = 0f;
        }

        public void GiveControlToPlayer()
        {
            agent.stoppingDistance = 0f;
            GameContext.SetPlayer(this);
        }
    }
}
Porque não usar desiredVelocity.magnitude no player

O NavMesh “empurra” a velocity e ela oscila (path corners, acceleration). Para blend walk/run o input (MoveAmount) é mais estável e responde no frame em que empurras o stick — a animação não atrasa atrás do agent.

4. ThirdPersonCamera — yaw proporcional

Assets/Scripts/Camera/ThirdPersonCamera.cs
namespace AF
{
    using UnityEngine;

    /// <summary>
    /// Pitch/distância fixos. Yaw acompanha o personagem —
    /// mais depressa quando o stick está a fundo, mais lento com pressão leve.
    /// </summary>
    public class ThirdPersonCamera : MonoBehaviour
    {
        [Header("Target")]
        [SerializeField] CharacterManager target;
        [SerializeField] float targetHeight = 1.5f;

        [Header("Framing (fixo)")]
        [SerializeField] float distance = 5f;
        [SerializeField] float pitch = 25f;

        [Header("Yaw follow (analógico)")]
        [Tooltip("Yaw follow com stick leve / parado.")]
        [SerializeField] float yawFollowSpeedMin = 2.5f;
        [Tooltip("Yaw follow com stick a fundo.")]
        [SerializeField] float yawFollowSpeedMax = 10f;
        [Tooltip("Multiplicador extra com lock-on.")]
        [SerializeField] float lockOnYawMultiplier = 2f;

        [Header("Position smoothing")]
        [SerializeField] float followSmoothMin = 8f;
        [SerializeField] float followSmoothMax = 16f;

        [Header("Collision")]
        [SerializeField] LayerMask wallLayer;
        [SerializeField] float sphereRadius = 0.3f;
        [SerializeField] float wallBuffer = 0.15f;
        [SerializeField] float minDistanceFromTarget = 1.5f;

        float yaw;

        void Start()
        {
            yaw = target != null
                ? target.transform.eulerAngles.y
                : transform.eulerAngles.y;
        }

        void LateUpdate()
        {
            if (target == null) return;

            float moveAmount = target.MoveAmount; // 0..1

            UpdateYaw(moveAmount);

            Quaternion rotation = Quaternion.Euler(pitch, yaw, 0f);
            Vector3 pivot = target.transform.position + Vector3.up * targetHeight;

            Vector3 desiredPos = pivot - rotation * Vector3.forward * distance;
            desiredPos = ResolveWallOcclusion(desiredPos);

            float follow = Mathf.Lerp(followSmoothMin, followSmoothMax, moveAmount);
            if (target.lockOn != null && target.lockOn.isLockedOn)
                follow *= lockOnYawMultiplier;

            transform.position = Vector3.Lerp(
                transform.position,
                desiredPos,
                follow * Time.deltaTime
            );

            transform.rotation = Quaternion.LookRotation(
                pivot - transform.position, Vector3.up);
        }

        void UpdateYaw(float moveAmount)
        {
            float targetYaw = target.transform.eulerAngles.y;

            float yawSpeed = Mathf.Lerp(yawFollowSpeedMin, yawFollowSpeedMax, moveAmount);

            if (target.lockOn != null && target.lockOn.isLockedOn)
                yawSpeed *= lockOnYawMultiplier;

            // Mesmo parado, um follow mínimo evita a câmara ficar desalinhada para sempre
            // depois de um dodge / hit. Se quiseres câmara 100% congelada em idle,
            // early-return quando moveAmount < epsilon.
            yaw = Mathf.LerpAngle(yaw, targetYaw, yawSpeed * Time.deltaTime);
        }

        Vector3 ResolveWallOcclusion(Vector3 desiredPos)
        {
            Vector3 castOrigin = target.transform.position + Vector3.up * targetHeight;
            Vector3 direction = desiredPos - castOrigin;
            float maxDistance = direction.magnitude;
            if (maxDistance < 0.001f) return desiredPos;

            direction /= maxDistance;

            if (Physics.SphereCast(
                castOrigin,
                sphereRadius,
                direction,
                out RaycastHit hit,
                maxDistance,
                wallLayer,
                QueryTriggerInteraction.Ignore))
            {
                float pullDistance = Mathf.Max(
                    hit.distance - wallBuffer,
                    minDistanceFromTarget
                );
                return castOrigin + direction * pullDistance;
            }

            return desiredPos;
        }
    }
}
Feel

Stick leve → personagem anda devagar e a câmara “atrasa” um pouco (mais cinematográfico / controlável). Stick a fundo → corre e a câmara cola-se ao yaw. Em lock-on, o multiplicador mantém o tracking apertado.

5. Animator / Blend Tree

Para o meio-termo aparecer visualmente, o Blend Tree de locomoção tem de estar em 0..1 (não em m/s brutos).

  1. Parâmetro Speed (float).
  2. Blend Tree 1D:
    • 0.0 → Idle
    • 0.4 → Walk
    • 1.0 → Run
  3. Thresholds aproximados: Idle 0 · Walk 0.35–0.5 · Run 1.
Se o teu tree ainda usa m/s

Antes tinhas desiredVelocity.magnitude (~0–5). Agora mandamos 0..1. Ou reajustas os thresholds do Blend Tree para 0 / 0.4 / 1, ou no código fazes speedParam * runSpeed e manténs o tree antigo. Recomendo reajustar o tree — fica independente do valor de runSpeed.

// Opção B — manter thresholds em m/s no Animator:
float speedParam = Mathf.Lerp(walkSpeed, runSpeed, MoveAmount);
animator.SetFloat(SpeedHash, speedParam, animSpeedDamp, Time.deltaTime);
// Tree: Idle@0, Walk@walkSpeed, Run@runSpeed
